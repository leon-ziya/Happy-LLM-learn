# 设计规范（必须严格遵守）

## 整体风格
- 深色主题，主背景 `#070b14`，卡片背景 `#0f1724`，边框 `#1a2540`
- Canvas 粒子网络动画背景，粒子颜色跟随主色系，带连线效果
- 字体：标题/正文用 Google Fonts `Noto Sans SC`，代码/数据用 `JetBrains Mono`
- 图标用 Font Awesome 6
- 全部代码写在单个 HTML 文件中，CSS 用 `<style>` 标签，JS 用 `<script>` 标签
- 代码注释用中文

## 配色体系（三色原则）
根据章节主题选择三种配色角色，避免使用蓝紫默认配色：
- **主色（Primary）**：核心概念/主要流程 → 用高饱和度亮色，如 `#00e5a0`、`#f59e0b`、`#ef4444` 等
- **辅色（Secondary）**：次要概念/对比维度 → 用与主色互补或对比的亮色
- **第三色（Tertiary）**：辅助元素/输入数据 → 用偏冷或偏暖的中间色，如 `#38bdf8`、`#a78bfa`

在 CSS 变量中定义：
```css
:root {
  --bg:#070b14; --fg:#e8ecf4; --muted:#6b7a94;
  --primary:#___; --secondary:#___; --tertiary:#___;
  --card:#0f1724; --border:#1a2540;
}
```

## 排版规范
- 页面最大宽度 1280px，居中，内边距 20px-24px
- 标题 `clamp(28px, 4vw, 42px)`，字重 900
- 卡片圆角 16px，内边距 22-28px
- 按钮圆角 12px，悬停有发光/位移反馈
- 弹窗背景模糊 `backdrop-filter: blur(6px)`，入场动画 `translateY + scale`

# 页面结构（两模式切换）

## 模式一：【流程演示】

适用于有**明确流程/管线**的章节（如模型训练流程、数据处理流水线等）。

### 布局
```
┌─────────────────────────────────────────┐
│  标题 + 副标题                            │
├─────────────────────────────────────────┤
│  [流程演示]  [组件详解]   ← 模式切换按钮   │
├─────────────────────────────────────────┤
│  ● ─ ● ─ ● ─ ● ─ ●   ← 管道导航点      │
├──────────────────┬──────────────────────┤
│  架构/结构全景图  │  数据变换可视化         │
│  (左侧面板)      │  (右侧面板)            │
├──────────────────┴──────────────────────┤
│  步骤说明（编号 + 标题 + 详细文字）        │
├─────────────────────────────────────────┤
│  [← 上一步]           [下一步 →]          │
└─────────────────────────────────────────┘
```

### 管道导航
- 每个步骤一个圆点 + 标签，可点击跳转
- 当前步骤：圆点放大 + 发光 + 主色填充
- 已完成步骤：半透明主色填充
- 步骤间连线，完成后变为主色

### 左侧面板：结构全景图
- 用嵌套的 `div` 方块 + 箭头构建章节核心结构图
- 当前步骤对应的方块高亮（边框变色 + 发光 + 背景色变淡）
- 不同类别的方块用不同配色角色（primary / secondary / tertiary）

### 右侧面板：数据变换可视化
根据步骤类型选择合适的可视化方式（可组合使用）：

| 数据形态 | 推荐可视化 |
|---------|-----------|
| 文本/Token | 带颜色的圆角方块（类似 Token 盒子），入场有 translateY 动画 |
| 编号/ID | 同上，用 tertiary 色 |
| 向量/矩阵 | 条形图组（每个维度一个条），或热力图网格 |
| 注意力分数 | 网格热力图，数值映射为背景透明度，悬停显示具体值 |
| 掩码/遮挡 | 网格，可见=绿色✓，遮挡=红色✗ |
| 流程/管道 | 垂直排列的步骤方块 + 箭头，每步有入场延迟动画 |
| 残差/跳跃连接 | 垂直方块图 + 旁路虚线箭头标注 |
| 对比/分类 | 并排卡片，不同配色角色 |

### 步骤说明
- 编号圆圈（主色背景）+ 标题 + 段落
- 关键术语用 `<span class="highlight-text">` 标注，颜色对应其所属配色角色
- 代码片段用 `<code>` 包裹，背景淡主色

### 导航
- 上一步/下一步按钮，到头时 disabled
- 支持键盘左右箭头

## 模式二：【组件详解】

适用于有**多个并列知识点/组件**的章节。

### 布局
```
┌─────────────────────────────────────────┐
│  响应式网格，每张卡片约 280px 宽           │
│  ┌──────┐ ┌──────┐ ┌──────┐            │
│  │ 图标  │ │ 图标  │ │ 图标  │            │
│  │ 名称  │ │ 名称  │ │ 名称  │            │
│  │ 简介  │ │ 简介  │ │ 简介  │            │
│  └──────┘ └──────┘ └──────┘            │
└─────────────────────────────────────────┘
```

### 卡片设计
- 顶部 3px 渐变色条（hover 时显示）
- 图标区域：44x44 圆角方块，淡色背景 + 对应颜色图标
- 标题：16px，字重 900
- 简介：13px，muted 色
- 悬停：上移 3px + 边框变色 + 阴影

### 点击展开弹窗
弹窗内容按以下固定结构组织（每个 section 用 `<h3>` 分隔）：

1. **定义** → 图标 `fa-info-circle`
2. **组成与实现** → 图标 `fa-puzzle-piece`
3. **迷你可视化** → 放在 `.mini-viz` 容器中，复用流程演示中的可视化组件
4. **代码/公式** → 图标 `fa-code`，用 `<code>` 标签
5. **通俗理解** → 图标 `fa-lightbulb`，用比喻/类比

弹窗关闭方式：右上角关闭按钮 / 点击遮罩 / ESC 键

# 动画规范

| 元素 | 动画 |
|------|------|
| Token 盒子 | `translateY(12px)→0`，0.5s ease，依次延迟 |
| 向量条 | `scaleX(0)→1`，0.6s ease，依次延迟 |
| 卡片悬停 | `translateY(-3px)` + 边框发光 |
| 弹窗入场 | `translateY(20px) scale(.97)→0`，0.35s ease |
| 管道圆点 | 当前步骤 `scale(1.25)` + glow |
| 脉冲箭头 | opacity 0.5↔1，1.5s ease infinite |
