# 任务

请基于我提供的学习内容，生成一个**单文件 HTML 可视化交互页面**，帮助我直观理解该章节的核心知识点。

页面应像一张"知识地图"：既要有纵向的**发展脉络**（时间线 / 演进路径），也要有横向的**并列组件**（多个知识点对照），还要有底部的**学习路线与口诀**帮助形成长期记忆。

---

# 设计规范（必须严格遵守）

## 整体风格
- 深色主题，主背景 `#070b14`，卡片背景 `#0f1724`，边框 `#1a2540`
- Canvas 粒子网络动画背景：粒子颜色跟随主色系，带连线效果，鼠标附近粒子被吸引/连线增强
- 字体：标题/正文用 Google Fonts `Noto Sans SC`，代码/数据用 `JetBrains Mono`
- 图标用 Font Awesome 6
- 全部代码写在单个 HTML 文件中，CSS 用 `<style>` 标签，JS 用 `<script>` 标签
- 代码注释用中文
- 支持 `prefers-reduced-motion: reduce`，无障碍降级

## 配色体系（三色原则）
根据章节主题选择三种配色角色，**避免使用蓝紫默认配色**：
- **主色（Primary）**：核心概念 / 主要流程 / 主线演进 → 用高饱和度亮色，如 `#00e5a0`、`#f59e0b`、`#ef4444` 等
- **辅色（Secondary）**：次要概念 / 对比维度 / 次要分支 → 用与主色互补或对比的亮色
- **第三色（Tertiary）**：辅助元素 / 输入数据 / 第三类对象 → 用偏冷或偏暖的中间色

**色彩角色映射规则**：每个章节需明确把三色映射到该章的三个核心维度，例如：
- 第一章 NLP：primary=NLP 概念主线 / secondary=NLP 任务（理解类）/ tertiary=NLP 任务（生成类）
- 第三章 PLM：primary=Encoder-Only / secondary=Decoder-Only / tertiary=Encoder-Decoder

在 CSS 变量中定义：
```css
:root {
  --bg:#070b14; --fg:#e8ecf4; --muted:#6b7a94;
  --primary:#___; --secondary:#___; --tertiary:#___;
  --card:#0f1724; --border:#1a2540;
  /* 语义化辅助色 */
  --ok:#10b981;     /* 正确/优点/正例 */
  --err:#ef4444;    /* 错误/缺点/负例 */
  --warn:#fbbf24;   /* 警告/中性/折中 */
}
```

## 排版规范
- 页面最大宽度 1280px，居中，内边距 20px-24px
- 标题 `clamp(26px, 4vw, 40px)`，字重 900，可用渐变 `-webkit-background-clip: text` 给主标题着色
- 副标题 14-15px，muted 色
- 卡片圆角 16px，内边距 22-28px
- 按钮圆角 12px，悬停有发光/位移反馈
- 弹窗背景模糊 `backdrop-filter: blur(6px)`，入场动画 `translateY + scale`

# 页面结构（三段式 + 双模式）

整体页面分**三段**：顶部 Hero / 中部双模式主体 / 底部总结与学习路线。

```
┌─────────────────────────────────────────────────────────┐
│  Hero 段：章节徽章 + 主标题 + 副标题 + 章节概览四宫格       │
├─────────────────────────────────────────────────────────┤
│  [模式一] [模式二] [模式三*]    ← 模式切换按钮              │
├─────────────────────────────────────────────────────────┤
│  主体：根据当前模式渲染（详见下文）                          │
├─────────────────────────────────────────────────────────┤
│  Footer 段：核心总结 + 学习路线口诀 + 章节导航提示           │
└─────────────────────────────────────────────────────────┘
```

*模式三为可选，仅当章节同时存在「流程」「组件」之外的第三个独立维度时启用（如"对比矩阵"或"实战演练"）。

## Hero 段（顶部章节概览）
- 章节徽章：圆角胶囊形状，主色淡背景，主色文字与边框，如 `第一章`
- 主标题：渐变着色，字重 900
- 副标题：一句话总览本章，muted 色
- **章节概览四宫格**（必备）：在副标题下方，4 个小卡片横向排列，每张 1 个图标 + 1 个数字/关键词 + 1 个标签
  - 例：`📖 9 个核心任务` / `⏳ 80 年发展史` / `🧠 4 代文本表示` / `🎯 3 大主线`
  - 用于让读者 3 秒内把握章节体量与维度

## 模式一：【流程演示 / 发展脉络】

适用于有**明确时间线 / 流程 / 演进路径**的章节（如 NLP 发展史、模型训练流程、文本表示演进等）。

### 布局
```
┌─────────────────────────────────────────┐
│  标题 + 副标题                            │
├─────────────────────────────────────────┤
│  ● ─ ● ─ ● ─ ● ─ ●   ← 管道导航点      │
│  (含 era 标签，如 1940s / 2013 / 预训练) │
├──────────────────┬──────────────────────┤
│  演进时间线        │  数据变换可视化         │
│  (左侧面板)       │  (右侧面板)            │
│  - 垂直时间线      │  - 每步配套不同可视化   │
│  - 进度填充动画    │  - 支持多种可视化形态   │
│  - 过去节点淡色    │                       │
│  - 当前节点高亮    │                       │
├──────────────────┴──────────────────────┤
│  步骤说明（编号 + 标题 + 详细文字 + 关键统计）│
├─────────────────────────────────────────┤
│  [← 上一步]   [进度 N/M]   [下一步 →]    │
└─────────────────────────────────────────┘
```

### 管道导航
- 每个步骤一个圆点 + 标签，**额外可显示 era 副标签**（如 `1940-1960s`、`2013`、`预训练`）
- 当前步骤：圆点放大 `scale(1.3)` + 发光 + 主色填充
- 已完成步骤：半透明主色填充
- 步骤间连线，完成后变为主色
- 支持点击任意节点跳转

### 左侧面板：演进时间线（推荐使用垂直时间线）
垂直时间线结构（比纯方块图更适合"演进"主题）：
```
│
●── 1940-1960s · 早期探索
│   图灵测试、机器翻译萌芽
│
●── 1970-1990s · 符号与统计
│   规则 vs 概率，统计模型崛起
│
●── 2000s+ · 深度学习时代   ← 当前节点高亮
│   RNN→LSTM→Attention→Transformer
│
○── 未来节点（淡色未激活）
│
```
- 一条垂直主轴 + 各节点圆点
- 主轴有**进度填充**动画（已走过的部分为主色，未走过的为 border 色）
- 当前节点：圆点主色 + 发光 + 名称变主色 + 描述展开
- 过去节点：圆点淡主色填充
- 未来节点：透明度 0.3，仅显示名称

若章节为"结构图"而非"时间线"（如 Transformer 架构），可改用**嵌套 div 方块 + 箭头**的结构全景图，规则同原模板。

### 右侧面板：数据变换可视化（**根据步骤类型自动切换**）

下表为推荐可视化形态，**每步可组合多种**：

| 数据形态 | 推荐可视化 | 实现要点 |
|---------|-----------|---------|
| 文本/Token | 彩色圆角方块（Token 盒子） | `translateY` 入场，按词延迟 0.05s |
| 编号/ID | 同上，用 tertiary 色 | 等宽字体显示数字 |
| 稀疏向量（One-Hot） | **稀疏条形图**：16 条细条，仅 1 条满高亮，其余 2% 高度+30% 透明度 | 配数据"稀疏率 99.97%" |
| 稠密向量 | 横向条形图组，每条随机宽高+不同透明度 | 配标签"低维稠密 / 几百维" |
| 向量空间 | **散点空间图**：相对定位的彩色圆点，按类别聚类 | 不同类别用不同配色角色，含图例 |
| 注意力分数 | 网格热力图 | 数值映射背景透明度，悬停显示具体值 |
| 掩码/遮挡 | 网格，可见=绿色✓，遮挡=红色✗ | 用于 BERT MLM / GPT 因果掩码 |
| 概率链（N-gram） | **N-gram 演示卡**：词盒 → 词盒 → 概率值 | Uni/Bi/Tri-gram 用三色区分 |
| 流程/管道 | 垂直步骤方块 + 脉冲箭头 | 每步入场延迟 0.1s |
| 演进路径 | **水平箭头流程**：阶段1 → 阶段2 → 阶段3 → 阶段4 | 阶段方块用渐变色递进 |
| 残差/跳跃连接 | 垂直方块图 + 旁路虚线箭头 | 标注"残差"标签 |
| 对比/分类 | **对比面板**：左右两栏，good/bad 配色 | 一栏优点（绿）/一栏缺点（红） |
| 一词多义 | **语境对比卡**：同词汇在不同语境下向量不同 | 两个迷你向量条并排，颜色不同 |
| 关系三元组 | 主语 → 关系标签 → 宾语 | 三个胶囊，关系用 tertiary 色 |
| 实体标注 | 高亮文本 + 上标标签 | 不同实体类别用不同色 |
| 词性标注 | 单词 + 词性缩写标签（垂直堆叠） | 名词/动词/形容词用不同色 |
| 任务分级 | 卡片网格 + 等级标签 | 基础/核心/高级 三色标签 |

### 步骤说明（位于双栏下方）
- 编号圆圈（主色背景，深色文字）+ 标题 + 详细段落
- 关键术语用 `<span class="hl-p">` / `<span class="hl-s">` / `<span class="hl-t">` 标注，颜色对应配色角色
- 代码片段用 `<code>` 包裹，背景淡主色
- **关键统计行**（推荐）：段落末尾追加 1-3 个胶囊状数据标签，如 `词表 16384 / 稀疏率 99.97% / 维度灾难`
- 段落需丰富（150-250 字），不能单句收尾

### 导航
- 上一步/下一步按钮，到头时 disabled
- 中间显示**进度指示**：`3 / 8` 或进度条
- 支持键盘 ←/→ 切换
- 最后一步按钮文字变为"已完成"

### 卡片设计
- 顶部 3px 渐变色条（hover 时显示 opacity 1，默认 0）
- 图标区域：42×42 圆角方块，淡色背景 + 对应颜色图标
- 标题：15-16px，字重 900
- 简介：12-13px，muted 色，1-2 行
- **分级标签**（推荐）：右上角或底部胶囊标签，区分知识点等级
  - `基础` / `核心` / `高级` 三档，分别对应 primary / secondary / tertiary 色
  - 或按章节自定义，如 Encoder-Only / Decoder-Only / Encoder-Decoder
- 悬停：上移 3px + 边框变色 + 阴影

### 点击展开弹窗（**5 段固定结构**）
弹窗内容按以下固定结构组织（每个 section 用 `<h3>` + 对应图标分隔）：

1. **定义** → 图标 `fa-info-circle`，主色标题
   - 一段完整定义（80-150 字），点明"是什么 / 解决什么问题"
2. **组成与实现** → 图标 `fa-puzzle-piece`，辅色标题
   - 用 `<ul>` 列出 3-6 个组成要素或实现步骤
   - 每条 1-2 句话，含具体技术名词
3. **交互演示 / 迷你可视化** → 图标 `fa-play-circle`，第三色标题
   - 放在 `.mini-viz` 容器中，**复用流程演示中的可视化组件**
   - 必须有真实数据样例（如分词结果、NER 标注、向量空间等），不能空泛
4. **代码 / 公式** → 图标 `fa-code`，主色标题
   - 用 `<pre class="code-block">` 包裹，等宽字体
   - 含输入输出示例、关键参数、对比表格等
5. **通俗理解** → 图标 `fa-lightbulb`，辅色标题
   - 用 1-2 个生活化比喻/类比解释
   - 例："分词就像把没有空格的珠子按意义切割成组"

弹窗关闭方式：右上角关闭按钮 / 点击遮罩 / ESC 键

# 动画规范

| 元素 | 动画 |
|------|------|
| Token 盒子 | `translateY(10-12px)→0`，0.5s ease，依次延迟 0.05s |
| 向量条 | `scaleX(0)→1`，0.6s ease，依次延迟 |
| 稀疏条形 | 同向量条，但只有 1 条满高亮，其余保持 2% 宽 + 低透明度 |
| 时间线进度 | `height: 0% → N%`，0.5s ease，跟随当前步骤 |
| 卡片悬停 | `translateY(-3px)` + 边框发光 + 顶部色条显现 |
| 弹窗入场 | `translateY(18-20px) scale(.97) → 0`，0.35s ease |
| 管道圆点 | 当前步骤 `scale(1.25-1.3)` + glow |
| 脉冲箭头 | opacity 0.5↔1，1.5s ease infinite |
| 散点入场 | `translateY + opacity`，依次延迟 0.08s |
| 模式切换 | `opacity 0→1` + `translateY(10px)→0`，0.4s ease |

所有动画在 `prefers-reduced-motion: reduce` 下降级为 0.01ms。

# 内容深度规范（**强制**）

## 段落深度
- 每个步骤说明段落 ≥ 150 字，禁止单句段落
- 必须含具体数据 / 年份 / 参数量 / 模型名等可量化信息
- 关键术语必须用 `<span class="hl-*">` 高亮

## 弹窗内容深度
- 定义段：80-150 字，含"是什么 + 解决什么问题"
- 组成与实现：≥ 3 条要点，每条 ≥ 1 句话
- 交互演示：必须有真实数据样例（不能是占位符）
- 代码/公式：≥ 5 行有效内容
- 通俗理解：≥ 1 个比喻，30-80 字

## 知识点覆盖度
- 章节中提到的所有 NLP 任务 / 模型 / 技术名词都应在页面中有所体现
- 不可遗漏章节明示的核心知识点
- 同一知识点的不同表述需建立交叉引用（如"向量空间模型 VSM" 与 "One-Hot"）
